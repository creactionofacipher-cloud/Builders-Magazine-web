# Builders Magazine

## Project Overview

Builders Magazine is an independent printed magazine dedicated to custom motorcycle culture.

The goal of this project is to create a digital platform that extends the printed publication into the online space. The website should not replicate the magazine but complement it with unique digital content, archives, stories, and information about the community surrounding Builders Magazine.

The platform should emphasize visual storytelling, photography, craftsmanship, and the people behind custom motorcycles.

---

# Project Vision

The website should become the central digital platform for Builders Magazine.

It must combine several roles:

- official website of the magazine

- archive of printed issues

- digital magazine with exclusive online stories

- platform for Builders Cup

- entry point into the Builders Magazine community

The website should feel like a premium editorial publication rather than a news portal or online store.

---

# Project Goals

The platform should achieve the following objectives.

## 1. Present Builders Magazine

Introduce visitors to the magazine:

- what Builders Magazine is

- editorial philosophy

- team

- contacts

- cooperation opportunities

---

## 2. Showcase Printed Issues

Every issue should have its own dedicated page containing:

- cover

- release information

- description

- table of contents

- featured stories

- purchase links

The printed magazine remains the primary product.

The website serves as its digital companion.

---

## 3. Publish Digital Stories

The website should contain exclusive digital content that complements the printed publication.

Examples:

- custom motorcycles

- builders

- interviews

- workshops

- travel

- events

- motorcycle culture

- behind-the-scenes stories

The website should not duplicate magazine articles.

Instead, it should expand the Builders Magazine universe.

---

## 4. Build Community

The platform should connect people around Builders Magazine.

Examples:

- builders

- workshops

- photographers

- artists

- participants

- motorcycle projects

Future versions may include additional community features.

---

## 5. Represent Builders Cup

The website should become the official online platform for Builders Cup.

Each event should include:

- event information

- participants

- winners

- galleries

- related stories

Future versions may include registration and ticket purchasing.

---

# Target Audience

The primary audience consists of people interested in custom motorcycle culture.

Examples:

- motorcycle builders

- custom motorcycle owners

- photographers

- designers

- artists

- motorcycle enthusiasts

- readers of Builders Magazine

- Builders Cup visitors

The website should appeal to both existing readers and people discovering Builders Magazine for the first time.

---

# MVP Scope

The first version focuses on creating a solid foundation for future development.

Included in MVP:

- Home

- Magazine

- Issue pages

- Stories

- Story pages

- Builders Cup

- About

- Search

- Buy (Magazine & Merchandise)

Not included:

- shopping cart

- payment system

- user accounts

- authentication

- personal profiles

- community features

The architecture must allow these features to be added later without major refactoring.

---

# Product Principles

The project should follow several core principles.

## Editorial First

Content comes before interface.

Photography and stories are the main focus.

---

## Magazine Experience

The website should feel like reading a premium magazine.

Large imagery.

Strong typography.

Generous spacing.

Minimal distractions.

---

## Photography First

Photography is the primary visual language.

Layouts should be designed around images rather than text.

---

## Performance Matters

Fast loading.

Optimized images.

Minimal JavaScript.

Excellent Core Web Vitals.

---

## Scalable Architecture

The MVP is only the beginning.

Every architectural decision should support future growth.

---

## CMS Driven

Content should never be hardcoded.

Everything should be manageable through a headless CMS.

---

# Long-Term Vision

The long-term goal is to evolve Builders Magazine into a complete digital platform.

Future capabilities may include:

- Builder profiles

- Bike database

- Full online shop

- Builders Cup registration

- Ticket sales

- Interactive maps

- Community features

- Newsletter

- Multi-language support

- Mobile application

The MVP should provide the architectural foundation for this evolution.

---

# Success Criteria

The project will be considered successful if it achieves the following:

- represents Builders Magazine professionally

- showcases printed issues beautifully

- supports digital storytelling

- provides an excellent reading experience

- is easy to manage through a CMS

- performs well on desktop and mobile

- is ready for future expansion

---

# Project Status

Current phase:

Planning and MVP development.

The project will be implemented using a modern headless architecture with Next.js, TypeScript, Tailwind CSS, Sanity CMS, and Vercel.  
  
# What This Website Is Not

The website is NOT:

- a traditional news portal

- a motorcycle marketplace

- an online forum

- a blog

- a corporate website

Instead, it is a carefully curated editorial platform built around stories, photography, craftsmanship, and the culture of custom motorcycles.
