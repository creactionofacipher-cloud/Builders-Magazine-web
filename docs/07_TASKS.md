# **Development Rules**

During implementation, follow these principles:

- Complete one milestone before starting the next.
- Keep components small and reusable.
- Avoid duplicate code.
- Prefer Server Components.
- Maintain strict TypeScript typing.
- Keep business logic outside UI components.
- Test every completed feature.
- Commit changes frequently with meaningful commit messages.

Every completed milestone should leave the project in a stable, deployable state.
